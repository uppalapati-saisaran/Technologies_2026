Special Permissions 

SUID - Run program with owner privileges.
SGID - Files inherit group from directory.
Stick Bit - Only file owner can delete files.

1, SUID 
touch script.sh; chmod u+s script.sh, ls -l script.sh 
_rwsr_xr_r - Notice ths s in owner execute postion.
           - Meaning: user executing this file runs it as the file owner 
Example real command using SUID:
/usr/bin/passwd - the passwd command runs as root even when normal users run it. 

2, SGID 
mkdir devteam; chmod g+s devteam; ls -ld devteam 
_rwxrwsr_r - All new files inside the directory inherit the group.
Example: devteam/file1 devteam/file2 - Both automatically belong to devteam group.

3, Sticky Bit 
Used for shared directories 

/tmp; ls -ld /tmp; 
drwxrwxrwt - Users can only delete their files.

Exam Task:  Create shared directory where users can create file sbut cannot delete other's files.

mkdir /shared 
chmod 1777 /shared 
ls -ld /shared 
drwxrwxrwt

Permission   value 
SUID          4
SGID          2
Sticky Bit    1 
Example chmod 4755 file 
Meaning : 4 -> SUID, 755 -> normal permissions.
